run_test (workload, opts)
Create test and run.

Parameters:

  • workload Table with workload options.
    • create_client Function that creates a workload client.
    • generator table Generator of operations used in workload. Generator must be made using Lua functional library or jepsen.gen module.
    • checker Function for checking history in workload. (optional)
  • opts Table with test options.
    • logging boolean Option to enable logging history to files: history.txt with plain history, and history.json with history encoded to JSON. (optional)
    • threads number Number of threads in workload, default value is 1. (optional)
    • thread_type string Type of threads used in workload. Possible values are 'fiber' and 'coroutine', default value is 'fiber' on Tarantool and 'coroutine' on LuaJIT. (optional)
    • time_limit number Number of seconds to limit time of testing. By default testing timeis endless and limited by a number of operations produced by generator. (optional)
    • nodes table A table that contains IP addresses of nodes participated in testing. (optional)

Returns:

    {}

See also:

    jepsen.gen

Usage:

    local test_options = {
    logging = true,
    thread_type = 'fiber',
    threads = 5,
    nodes = {
        '127.0.0.1'
    }
}
local ok, err = jepsen.run_test({
    create_client = new,
    generator = gen_lib.cycle(gen_lib.iter({ r, w })):take(1000)
}, test_options)
